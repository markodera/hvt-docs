# Security Policy

## Supported Versions

Security fixes are applied to the latest code on the default branch.

## Reporting a Vulnerability

Do not open public GitHub issues for security reports.

Use GitHub private vulnerability reporting for this repository if it is enabled. If private reporting is unavailable, contact the project maintainers through the contact details published at [hvts.app](https://hvts.app/) and include:

- A clear description of the issue
- Reproduction steps or a proof of concept
- The affected files, routes, or deployment path
- Any suggested mitigation if you have one

We aim to acknowledge reports within 5 business days and will coordinate remediation and disclosure with the reporter.
